# FormStream 前期准备清单

> 配合 `FormStream-开发文档.md`（§7 数据模型、§12 部署配置）使用。这份清单只列**你现在需要去 Supabase / Cloudflare 控制台（或 CLI）手动做的事**，不涉及写代码。做完后把关键值（URL、key、id）记下来发给我，我们就可以正式开工。

---

## 一、Supabase 准备事项

### 1. 建项目
- [ ] 打开 supabase.com → New Project
- [ ] 项目名建议 `formstream`（如果想区分环境，可以建 `formstream-dev` / `formstream-prod` 两个项目） formstream-dev
- [x] 数据库密码：随机生成一个强密码，保存好（密码管理工具），不要用弱密码——这是 Postgres 超级用户密码（已生成，存在密码管理工具里，不要再写进任何项目文件）
- [ ] Region：选离你目标用户最近的区域即可（项目定位是海外站，选 us-east-1 或新加坡都可以，影响不大，后续不好迁移所以选好一点）
- [ ] Plan：开发阶段先用 **Free** 没问题；**但正式上线前必须升级到 Pro**——Free 档闲置 7 天会自动暂停项目，不适合生产环境（文档 §11.2 已提到）

### 2. 初始化数据库（在 SQL Editor 里执行）
- [ ] 执行开发文档 **§7.2 DDL**：建表 `profiles` / `forms` / `notification_channels` / `submissions` / `usage`，以及 `handle_new_user` 触发器
- [ ] 执行开发文档 **§7.3 RLS**：给 `profiles` / `forms` / `submissions` / `notification_channels` 开启 RLS 并建策略

> 这两步可以现在就在 Supabase 控制台手动跑一遍把库建起来；后面如果需要改表结构，我们再一起处理 migration。

### 3. Auth 配置
- [ ] Authentication → Providers：确认 **Email** 登录方式已启用（默认就是开的，确认一下即可）
- [ ] Authentication → URL Configuration：
  - Site URL：先填你的 `*.workers.dev` 子域名（还没有正式域名也没关系，部署后能拿到），后面换了自定义域名再回来改
  - Redirect URLs：先留空或填同一个域名，等我们做注册/找回密码邮件跳转时再精确配置
- [ ] 邮件模板：先用 Supabase 自带的默认模板（确认邮件、找回密码邮件）即可，不用现在折腾自定义 SMTP

### 4. 记录关键值（Settings → API）
- [ ] `Project URL`（形如 `https://xxxx.supabase.co`）
- [ ] `anon public` key
- [ ] `service_role` key —— **注意：这是最高权限密钥，绕过 RLS，绝对不能进代码仓库或下发到前端**，只会作为 Cloudflare Worker 的 secret 存在

### 5. 想好第一个管理员邮箱
- [ ] 确定你自己要用哪个邮箱作为平台的第一个管理员账号（后面会配置成 Cloudflare secret `ADMIN_EMAILS`，用这个邮箱注册登录后会自动被提升为 `admin`）

## 二、Cloudflare 准备事项

### 1. 账号与计费
- [ ] R2 / KV 首次启用通常需要账号绑定支付方式（即使在免费额度内使用，也可能要求验证过支付方式），先去 Dashboard 绑一下
- [ ] **Cloudflare Queues 免费计划也能用**（之前文档写的"必须升级 Paid"是过时信息，已核实更正）：
  - Free 计划：每天 10,000 次 operations 免费，消息保留期固定 24 小时（不可调）
  - Paid 计划（$5/月起）：每月 1,000,000 次 operations 免费，消息保留期可配置到 14 天
  - 开发阶段直接用 Free 计划创建队列即可，不用现在升级；等通知量真的上来了（一天超 1 万次操作）或需要更长重试窗口时再考虑升级

### 2. 创建 R2 bucket（文件上传用）
- [ ] Dashboard → R2 → Create bucket，名字用 `formstream-uploads`（要和 wrangler 配置里的 `bucket_name` 对上）
- [ ] 或者用命令行：
  ```bash
  npx wrangler r2 bucket create formstream-uploads
  ```

### 3. 创建 KV namespace（限流/缓存用）
- [ ] 命令行创建：
  ```bash
  npx wrangler kv namespace create RATE_LIMIT
  ```
- [ ] 记下返回的 `id`，后面要填进 `wrangler.json` 的 `kv_namespaces`

### 4. 创建 Queue（通知派发用，需要 Paid 计划）
- [ ] 命令行创建：
  ```bash
  npx wrangler queues create formstream-notify
  ```

### 5. Turnstile（人机验证，防滥用）
- [ ] Dashboard → Turnstile → Add site
- [ ] Domain 先填你打算用的开发/正式域名（没有正式域名可以先随便填一个占位，后面再改）
- [ ] 拿到并记录：
  - `Site Key`（前端表单页要用，不是机密）
  - `Secret Key`（后端校验用，要配置成 secret `TURNSTILE_SECRET`）

### 6. 自定义域名（如果已经有域名）
- [ ] 如果域名还没托管在 Cloudflare，先把域名加到 Cloudflare（NS 切过来）
- [ ] Worker → Settings → Domains & Routes → 绑定自定义域名
- [ ] 暂时没有域名也不影响开发，先用默认的 `*.workers.dev` 子域名顶上，后面再换

### 7. Resend（邮件通知渠道）
- [ ] 注册 resend.com 账号
- [ ] Domains → 添加你要用来发信的域名，按提示在 DNS 里加 SPF / DKIM 记录（如果域名已经托管在 Cloudflare，直接在 Cloudflare DNS 里加这些记录就行）
- [ ] 拿到并记录 `RESEND_API_KEY`

### 8. 汇总你需要收集好的值
开发正式开始时，会用 `wrangler secret put` 把以下机密写进 Worker（现在不用执行，先把值收集好发我）：

| 名称 | 来源 | 说明 |
|---|---|---|
| `SUPABASE_URL` | Supabase Settings → API | 非机密，可作 var |
| `SUPABASE_ANON_KEY` | Supabase Settings → API | 验用户 JWT / SSR session |
| `SUPABASE_SERVICE_KEY` | Supabase Settings → API | **机密**，绕过 RLS |
| `RESEND_API_KEY` | Resend Dashboard | 发邮件 |
| `TURNSTILE_SECRET` | Cloudflare Turnstile | 校验人机验证 |
| `ADMIN_EMAILS` | 你自己定 | 逗号分隔，首个管理员邮箱 |

加上前面创建资源时拿到的：
- R2 bucket 名称（`formstream-uploads`）
- KV namespace id
- Queue 名称（`formstream-notify`）
- Turnstile Site Key（前端要用，非机密）
